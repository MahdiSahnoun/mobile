RAPPORT TECHNIQUE D'ANALYSE

My_Gym est une application mobile d'entraînement assistée par Intelligence Artificielle. Elle utilise la vision par ordinateur en temps réel pour compter les répétitions d'exercices (Squats) et synchronise les performances des utilisateurs via Firebase Realtime Database.

1. Vue d'ensemble du Projet

L'architecture suit un modèle modulaire découplant l'interface utilisateur, la logique métier (services) et le rendu graphique (painters).

2. ML Kit — Analyse Approfondie

SERVICE : Pose Detection

Le projet utilise exclusivement le service Google ML Kit Pose Detection. Ce service utilise un réseau de neurones profonds pour identifier 33 points clés (landmarks) du corps humain.

Fonctionnement Technique

Le détecteur traite un objet InputImage et retourne une liste de Pose. Chaque pose contient des PoseLandmark avec des coordonnées (x, y) et un score de confiance.

// Configuration du service dans pose_deection_service.dart
final options = PoseDetectorOptions(
  mode: PoseDetectionMode.stream, // Optimisé pour le flux vidéo continu
  model: PoseDetectionModel.accurate, // Précision maximale pour le calcul d'angles
);

Traitement des résultats

Les résultats sont envoyés au ExerciseCounterService pour la logique métier et au PosePainter pour l'affichage graphique.

3. Caméra — Logique Complète

PACKAGE : camera ^0.12.0

Le Pipeline Caméra

  1. Initialisation : Le CameraController est configuré avec ResolutionPreset.medium pour équilibrer fluidité et charge CPU.
  2. Stream d'Images : La méthode startImageStream capture les frames à environ 30 FPS.
  3. Format des Pixels : Sur Android, le format utilisé est NV21 (YUV420), optimisé pour ML Kit.

Conversion CameraImage → InputImage

C'est l'étape la plus critique techniquement. On extrait les bytes du premier plan de l'image (Y) pour les transformer en données exploitables par l'IA.

// Code clé de conversion dans pose_camera_screen.dart
return InputImage.fromBytes(
  bytes: image.planes[0].bytes, // Données brutes de la frame
  metadata: InputImageMetadata(
    size: Size(image.width.toDouble(), image.height.toDouble()),
    rotation: rotation, // Rotation basée sur le capteur (ex: 270°)
    format: format, // NV21 ou BGRA8888
    bytesPerRow: image.planes[0].bytesPerRow,
  ),
);

4. pose_painter.dart

Ce fichier gère la couche de Réalité Augmentée (AR) simplifiée en dessinant le squelette sur la preview.

Logique de Dessin

Il utilise canvas.drawLine pour relier les articulations selon une liste de connexions prédéfinies (épaules -> coudes -> poignets).

// Gestion de l'effet miroir (Caméra Frontale)
Offset _scale(PoseLandmark landmark, Size canvasSize) {
  final double x = landmark.x * canvasSize.width / imageSize.width;
  final double y = landmark.y * canvasSize.height / imageSize.height;
  
  // Inverse l'axe X si frontale pour que le dessin suive le mouvement utilisateur
  return Offset(isFrontCamera ? canvasSize.width - x : x, y);
}

6. Services & Logique Métier

ExerciseCounterService (Algorithme Squat)

Utilise la trigonométrie pour compter les répétitions.

// Calcul de l'angle du genou (Hanche-Genou-Cheville)
final angle = _calculateAngle(leftHip, leftKnee, leftAnkle);

if (angle < 90 && !_isDown) _isDown = true; // Phase descendante
if (angle > 160 && _isDown) { // Phase remontée validée
    _isDown = false;
    _repCount++;
}

8. Passages Critiques (Top 3)

1. UI Thread Management : L'utilisation du flag _isProcessing dans le stream caméra évite de saturer le processeur avec trop de frames simultanées.

2. StreamBuilder Architecture : Dans le Drawer, l'utilisation d'un Stream sur Firestore/RTDB garantit que la photo de profil est toujours synchronisée sans rechargement manuel.

3. Custom Coordinate Scaling : La logique dans PosePainter qui adapte les coordonnées ML Kit (basées sur la frame brute) à la taille réelle du widget Flutter à l'écran.
